# 链航黑匣：面向多部门协同的无人机可信飞行日志与事故追溯平台

## 一、项目名称

**链航黑匣：面向多部门协同的无人机可信飞行日志与事故追溯平台**

也可使用正式名称：

**面向低空安全治理的无人机可信飞行日志存证与多部门协同追溯平台**

---

## 二、项目定位

本项目面向低空经济快速发展背景下的无人机安全监管、事故追责和多部门协同需求，构建一个基于区块链的无人机可信飞行日志与事故责任追溯平台。

系统通过采集无人机飞行过程中的关键数据，包括飞行轨迹、姿态、电池状态、控制指令、任务状态、异常事件等信息，采用“链上存证 + 链下存储”的方式，将原始数据存储在链下系统，将日志哈希、时间戳、无人机身份、任务编号和数字签名写入区块链，从而保证飞行数据不可篡改、可验证、可追溯。

在此基础上，系统预留公安、消防、应急管理、低空监管平台等外部系统接口，实现异常事件推送、飞行身份核验、事故证据共享、应急任务协同和临时管控区同步，使平台从单一事故追责系统拓展为面向低空安全治理的多部门协同基础设施。

---

## 三、项目背景

随着无人机在低空物流、环境巡检、应急救援、城市治理、电力巡检、农业植保等领域的应用不断增加，低空空域中无人机数量快速增长。无人机规模化运行带来了新的安全治理问题，例如：

- 无人机身份难以快速识别；
- 飞行轨迹和任务记录可能被篡改；
- 事故发生后责任难以判定；
- 监管部门、运营企业、保险公司和制造商之间数据不互通；
- 公安、消防、应急等部门缺少统一的数据接口；
- 低空突发事件中缺少可信的多部门协同机制。

传统无人机飞行日志大多存储在本地设备、厂商云平台或企业数据库中，存在数据丢失、篡改、删除和多方互不信任的问题。一旦发生坠机、撞人、违规飞行、载荷丢失、进入禁飞区等事件，事故调查和责任认定往往缺少可信证据。

因此，本项目提出利用区块链的不可篡改、可追溯、可审计和多方共享特性，为无人机建立一个“可信黑匣子”，实现飞行过程记录可信、事故证据可信、外部系统调用可信、责任追溯可信。

---

## 四、要解决的核心问题

| 问题 | 具体表现 | 本项目解决方式 |
|---|---|---|
| 飞行身份不清 | 违规飞行、黑飞、闯入敏感区域时难以确认无人机和操作者 | 建立无人机链上身份和操作者身份凭证 |
| 飞行日志不可信 | 事故后本地日志可能被删除或修改 | 对关键日志进行哈希计算、数字签名和链上存证 |
| 事故责任难判定 | 难以判断事故由人为操作、硬件故障、算法异常还是环境因素导致 | 构建“任务—指令—状态—异常—结果”的完整证据链 |
| 多主体数据不互信 | 监管部门、运营商、保险公司和制造商各自掌握部分数据 | 使用联盟链实现多方可信共享 |
| 应急联动不足 | 公安、消防、应急等系统难以及时获取无人机异常信息 | 预留外部系统接口，实现事件推送和任务协同 |
| 数据调用不可追责 | 外部系统调取数据后缺少审计记录 | 将访问行为、授权记录和证据调取过程上链 |

## 五、系统总体架构

系统整体采用“无人机数据采集层—边缘可信处理层—链下存储层—区块链存证层—事故分析层—外部系统接口层—多部门应用层”的架构。

```text
无人机 / 机载电脑 / 飞控系统
        ↓
飞行数据采集层
        ↓
日志切片 + 哈希计算 + 数字签名
        ↓
链下存储层
        ↓
区块链存证层
        ↓
事故分析与责任追溯层
        ↓
外部系统接口层
        ↓
公安 / 消防 / 应急 / 低空监管 / 保险 / 运营商系统
```

## 六、核心功能模块

### 1. 无人机链上身份注册模块

为每架无人机建立唯一数字身份，记录无人机基础信息和运营资质。

主要数据包括：

- 无人机 ID；
- 机型与序列号；
- 制造商信息；
- 运营主体；
- 飞控编号；
- 操作者身份；
- 证书状态；
- 保险状态；
- 链上身份哈希。

该模块用于解决无人机“是谁的、谁在飞、有没有资质”的问题。

---

### 2. 飞行任务备案模块

每次飞行前生成任务记录，并将任务关键信息写入区块链。

主要内容包括：

- 任务编号；
- 飞行目的；
- 起点和终点；
- 批准航线；
- 飞行时间窗口；
- 操作者；
- 载荷类型；
- 风险等级；
- 监管授权状态。

该模块用于证明无人机任务是否经过备案，是否按照批准范围执行。

---

### 3. 飞行日志可信采集模块

系统通过飞控、机载电脑或通信接口采集无人机运行数据。

采集内容包括：

- GPS/RTK 坐标；
- 高度、速度、航向；
- 姿态角；
- 电池电压、电流、剩余电量；
- 飞行模式切换；
- 遥控器输入；
- 自动驾驶指令；
- 避障触发记录；
- 通信状态；
- 异常报警；
- 载荷状态；
- 任务完成状态。

如果无人机具备机械臂、夹爪或投送机构，还可以额外记录：

- 机械臂动作；
- 夹爪开合状态；
- 抓取成功或失败；
- 投放位置；
- 载荷重量变化；
- 视觉识别结果；
- 关键图像哈希。

---

### 4. 链上链下协同存储模块

由于飞行日志、视频、图像和点云数据体积较大，不适合全部直接上链，因此系统采用链上链下协同存储方式。

链下存储内容：

- 原始飞行日志；
- 视频文件；
- 图像文件；
- 点云数据；
- 传感器高频数据；
- 事故证据包原始文件。

链上存储内容：

- 日志哈希；
- 时间戳；
- 无人机身份；
- 任务编号；
- 事件编号；
- 存储地址索引；
- 数字签名；
- 证据包哈希；
- 访问授权记录。

这样既可以保证数据可信，又可以避免区块链系统负载过高。

---

### 5. 异常事件检测模块

系统在飞行过程中实时识别异常事件，并触发链上存证和外部系统推送。

异常事件包括：

- 低电量；
- GPS 丢失；
- 通信中断；
- 姿态异常；
- 碰撞预警；
- 强制接管；
- 越界飞行；
- 超高飞行；
- 禁飞区闯入；
- 急停；
- 坠落；
- 载荷脱落；
- 机械臂异常；
- 视觉识别失败。

系统可以按照两种方式进行存证：

1. **定时存证**：每隔 5 秒或 10 秒生成一次日志摘要并上链；
2. **事件存证**：发生异常事件时立即生成事件记录并上链。

---

### 6. 事故证据包生成模块

当系统检测到坠落、碰撞、失联、越界、急停、载荷脱落等事故时，自动提取事故前后关键数据，生成事故证据包。

事故证据包包括：

- 事故时间；
- 事故地点；
- 无人机身份；
- 操作者身份；
- 事故前后飞行轨迹；
- 飞行模式变化；
- 电池状态；
- 遥控器输入；
- 自动驾驶指令；
- 通信状态；
- 异常报警；
- 视频/图像索引；
- 链上存证编号；
- 日志完整性验证结果。

该模块是系统用于事故责任追溯的核心功能。

---

### 7. 事故责任分析模块

平台根据事故证据包和链上存证记录，对事故进行初步分析。

可能的责任类型包括：

- 操作者违规操作；
- 无人机硬件故障；
- 自动驾驶算法异常；
- 通信链路中断；
- GPS 或传感器异常；
- 任务规划错误；
- 第三方干扰；
- 环境突发因素；
- 不可抗力。

需要注意的是，系统输出的是辅助分析结果，最终责任判定仍应由监管部门、保险机构、司法机构或相关责任主体确认。

---

### 8. 可视化追溯平台

平台前端提供清晰的事故复盘与证据验证界面。

主要功能包括：

- 飞行轨迹回放；
- 关键事件时间轴；
- 链上存证记录查询；
- 日志完整性验证；
- 事故证据包下载；
- 多角色权限管理；
- 事故责任分析报告生成；
- 监管报告和保险理赔报告导出。

## 七、外部系统接口设计

为增强系统拓展性和实际应用价值，平台预留面向公安、消防、应急管理、低空监管、保险等外部系统的接口。

### 1. 接口设计原则

#### 1.1 不直接开放原始数据库

所有外部系统必须通过 API 网关调用接口，不能直接访问平台数据库。

这样可以实现：

- 访问可控；
- 权限可控；
- 调用留痕；
- 数据脱敏；
- 异常拦截；
- 访问频率限制。

#### 1.2 不开放全部原始数据

外部系统根据权限获取数据。普通查询只返回脱敏数据，涉及事故调查、违法取证、消防救援等场景时，才可在授权条件下调取完整证据包。

#### 1.3 不允许外部系统直接控制无人机

公安、消防、应急等系统可以发出：

- 协查请求；
- 空域预警；
- 应急任务请求；
- 临时禁飞区信息；
- 事故证据调取请求。

但不建议外部系统直接接管无人机。涉及返航、迫降、避让、任务中止等操作时，应通过平台授权机制、运营方确认机制和完整日志记录机制完成。

#### 1.4 所有访问行为上链审计

外部系统每一次数据访问、证据调取、任务发布和权限变更都应生成审计记录，并写入区块链。

记录内容包括：

- 调用系统；
- 调用时间；
- 调用接口；
- 调取数据类型；
- 调用理由；
- 关联事件编号；
- 授权状态；
- 返回结果哈希。

## 八、公安系统接口

公安系统主要关注违规飞行、黑飞治理、重大活动安保、人员安全、违法取证和事故调查。

| 接口名称 | 功能 | 典型用途 |
|---|---|---|
| 无人机身份核验接口 | 查询无人机注册身份、运营主体和证书状态 | 发现可疑无人机后快速识别 |
| 实时异常事件推送接口 | 推送越界飞行、失联、坠落、非法进入敏感区域等事件 | 公安指挥平台实时预警 |
| 飞行轨迹调取接口 | 查询指定任务的飞行轨迹和时间线 | 事故调查、违法取证 |
| 事故证据包接口 | 调取事故前后日志、视频哈希和事件链 | 责任认定、司法存证 |
| 临时管控区同步接口 | 接收公安设置的临时管控区域 | 大型活动、安保任务 |
| 调证审计接口 | 记录公安系统访问行为 | 防止数据滥用 |

### 示例接口

```http
GET /api/v1/public-security/uav/{uav_id}/identity
```

    "lat": 30.2741,

    "alt": 82.5
  },
  "blockchain_identity_hash": "0x8fa3..."
}
```

## 九、消防系统接口

消防系统主要关注火灾现场空域安全、无人机救援协同、灾情侦察、物资投送、现场禁飞管理和救援数据共享。

| 接口名称 | 功能 | 典型用途 |
|---|---|---|
| 火情区域同步接口 | 接收消防系统的火灾坐标和危险区域 | 自动生成临时低空管控区 |
| 救援任务发布接口 | 消防系统向无人机平台发布侦察或投送任务 | 火场侦察、药品或通信设备投送 |
| 现场空域状态接口 | 查询火场附近正在飞行的无人机 | 防止影响消防直升机或救援无人机 |
| 视频/图像证据接口 | 获取无人机灾情图像、热成像数据索引 | 辅助火情研判 |
| 应急优先级接口 | 标记消防任务为高优先级 | 调度无人机资源 |
| 任务完成回执接口 | 返回任务执行结果和链上存证 | 形成救援过程记录 |

### 示例接口

POST /api/v1/fire-rescue/emergency-task
```

### 示例返回

  "assigned_uav": "UAV-2026-003",
  "blockchain_record": "0x72ab...",

## 十三、智能合约设计

系统建议设计五类核心智能合约。

| 合约名称 | 作用 |
|---|---|
| UAVIdentityContract | 管理无人机链上身份 |
| MissionLogContract | 管理飞行任务和日志哈希 |
| EvidenceContract | 管理事故证据包 |
| AccessControlContract | 管理公安、消防、应急等系统访问权限 |
| EmergencyLinkageContract | 管理应急任务请求和处理记录 |

### 1. UAVIdentityContract

用于注册和管理无人机身份，包括无人机 ID、运营主体、证书状态、保险状态和设备状态。

### 2. MissionLogContract

用于记录飞行任务和日志存证信息，包括任务编号、日志哈希、时间戳、存储索引和数字签名。

### 3. EvidenceContract

用于管理事故证据包，包括事故编号、证据哈希、事故时间、事故地点、关联任务和验证状态。

### 4. AccessControlContract

用于管理不同外部系统的数据访问权限，包括授权范围、访问理由、调用记录和审计结果。

### 5. EmergencyLinkageContract

用于记录消防、应急等部门发起的任务请求，以及无人机平台的响应、执行和完成情况。

---

## 十四、数据权限分级

系统接入公安、消防、应急等外部部门后，必须设计数据分级机制，避免数据滥用和隐私泄露。

| 数据级别 | 数据内容 | 可访问对象 |
|---|---|---|
| 公开级 | 脱敏后的任务完成状态和统计数据 | 公众、普通客户 |
| 业务级 | 任务编号、飞行结果、交付证明 | 运营商、客户 |
| 监管级 | 飞行轨迹、无人机身份、异常事件 | 监管、公安、消防、应急 |
| 敏感级 | 原始视频、操作者身份、事故证据包 | 授权部门、司法机构、保险机构 |

系统原则：

- 链上只保存哈希和索引；
- 原始数据链下加密存储；
- 敏感数据按权限开放；
- 外部调用行为全部留痕；
- 证据访问过程上链审计。

## 十五、技术路线

### 第一阶段：无人机飞行数据采集

使用 PX4、ArduPilot、ROS、MAVLink 等方式读取飞控数据，采集 GPS、姿态、电池、速度、飞行模式、遥控器输入、任务状态等信息。

### 第二阶段：日志切片与哈希计算

将飞行数据按时间窗口切片，生成结构化 JSON 日志文件，并使用 SHA-256 等算法计算哈希值。

### 第三阶段：数字签名与链下存储

使用无人机私钥、运营商私钥或平台签名机制对日志摘要进行数字签名。原始日志、视频和图像存储在 IPFS、MinIO、云对象存储或本地服务器中。

### 第四阶段：区块链存证

使用联盟链系统将任务编号、日志哈希、时间戳、无人机身份、事件编号、数字签名和存储索引写入智能合约。

可选技术包括：

- Hyperledger Fabric；
- FISCO BCOS；
- 长安链；
- 以太坊私链；
- IPFS + 智能合约。

### 第五阶段：事故检测与证据包生成

当系统检测到异常事件时，自动提取事故前后关键数据，生成事故证据包，并将证据包哈希写入区块链。

### 第六阶段：外部系统接口开发

通过 API 网关、身份认证、权限控制和访问审计机制，开发公安、消防、应急、监管、保险等系统接口。

### 第七阶段：可视化平台开发

开发前端界面，实现飞行轨迹回放、事件时间轴展示、链上存证查询、日志完整性验证、事故报告生成和多部门联动管理。

## 十六、Demo 演示设计

建议 Demo 设计为一次完整的无人机任务事故追溯流程。

### Demo 场景

无人机执行一次巡检或投送任务，中途出现越界飞行、GPS 异常或碰撞事件。

### 演示步骤

1. 注册无人机链上身份；
2. 创建飞行任务并备案；
3. 模拟无人机起飞并产生飞行数据；
4. 每隔 5 秒生成一次日志哈希并上链；
5. 模拟 GPS 异常、低电量、偏航或碰撞事件；
6. 系统自动生成事故证据包；
7. 异常事件通过接口推送给公安或消防系统；
8. 平台回放飞行轨迹和事件时间轴；
9. 点击“验证日志”，系统显示数据未被篡改；
10. 修改本地日志后再次验证，系统显示哈希不一致；
11. 公安、消防或保险系统按权限调取证据包；
12. 系统生成事故责任分析报告。

### Demo 核心展示点

- 区块链不是控制无人机，而是证明飞行证据没有被篡改；
- 飞行日志可以被验证；
- 外部部门可以通过接口联动；
- 数据调取行为本身也可以被审计；
- 平台可以同时服务监管、事故追责和应急协同。

### 创新点五：多角色权限分级与隐私保护

AI 可以用于异常检测和事故原因辅助分析，区块链保证输入数据可信，形成“可信数据 + 智能分析 + 责任追溯”的闭环。

## 二十一、项目摘要

随着低空经济和无人机规模化应用的发展，无人机在物流配送、环境巡检、应急救援和城市治理等场景中的使用频率不断提高，但现有无人机飞行日志多存储于本地设备或中心化平台，存在数据易丢失、易篡改、多方难互信、事故责任难判定等问题。为解决上述问题，本项目拟设计一种基于区块链的无人机飞行日志、事故责任追溯与多部门协同联动平台。

系统通过采集无人机飞行轨迹、姿态、电池状态、控制指令、任务状态和异常事件等关键数据，采用链上链下协同存储方式，将原始日志存储于链下系统，并将日志哈希、时间戳、无人机身份、任务编号和数字签名写入区块链，实现飞行数据的不可篡改存证。事故发生后，平台可自动生成事故证据包，支持飞行轨迹回放、日志完整性验证、异常事件分析和责任追溯报告生成。

在此基础上，系统预留面向公安、消防、应急管理和低空监管平台的标准化接口，实现无人机身份核验、异常事件推送、飞行轨迹调取、事故证据共享、临时管控区同步和应急任务协同等功能。当外部系统调取相关数据时，平台依据权限生成脱敏或完整证据包，并将访问行为写入区块链，保证数据共享过程可追溯、可审计、不可篡改。

本项目可服务于无人机监管、保险理赔、低空物流、巡检执法、消防救援、公安安防和无人机制造商质量分析等场景，为低空经济安全运行提供可信数据基础。

## 二十二、核心表达

本项目的核心不是“用区块链控制无人机”，而是：

> **用区块链证明无人机飞行过程中的关键证据没有被篡改。**

系统主线可以概括为：

- **飞行前**：身份注册与任务备案；
- **飞行中**：关键日志实时存证与异常事件推送；
- **事故时**：自动生成事故证据包；
- **事故后**：轨迹回放、证据验证与责任追溯；
- **应急中**：公安、消防、应急等部门通过接口实现协同联动。

最终形成：

> **无人机负责执行任务，平台负责记录过程，区块链负责保证证据可信，外部接口负责支撑多部门协同。**
