# 开发任务拆分

## 阶段 1：Fabric 链码与账本模型

- 已完成无人机身份、任务日志、事故证据、访问审计、应急联动五类合约初版。
- 已完成复合键索引，避免基础查询强依赖 CouchDB 富查询。
- 下一步可补充背书策略、私有数据集合和更细的角色校验。

## 阶段 2：API 网关

- 已完成本地 mock 账本和 Fabric Gateway 两种模式。
- 已完成无人机注册、任务备案、日志存证、证据包、公安身份核验、消防任务、监管日志查询、保险证据调取接口。
- 下一步可接入 JWT、部门级 API key、限流、字段脱敏策略。

## 阶段 3：链下存储

- 当前 demo 使用本地 `data/offchain` 写入 JSON 文件，并将 `offchain://bucket/file` 作为链上索引。
- 下一步可替换为 MinIO、IPFS 或云对象存储。
- 视频、图像、点云等大文件只保存哈希和索引上链。

## 阶段 4：飞行数据模拟与事故证据包

- 已提供 `scripts/demo-flight.js` 串联演示流程。
- 下一步可加入 PX4/ArduPilot/MAVLink 模拟数据源。
- 事故证据包建议按 `事故前 30 秒 + 事故后 30 秒` 的窗口抽取关键日志。

## 阶段 5：可视化追溯平台

- 建议使用地图轨迹回放、事件时间轴、链上存证记录、证据验证结果四块作为第一版界面。
- 前端可以优先读取 API 网关，不直接连接 Fabric。
- 报告导出可先生成 JSON/PDF，再扩展为保险理赔报告和监管报告。

## 阶段 6：课程演示建议

1. 注册无人机链上身份。
2. 创建飞行任务备案。
3. 模拟定时日志上链。
4. 模拟碰撞/越界/GPS 异常事件。
5. 生成事故证据包并写入证据哈希。
6. 展示日志哈希验证成功。
7. 修改链下日志后展示验证失败。
8. 公安/消防/保险接口调取数据。
9. 展示访问审计记录已上链。
